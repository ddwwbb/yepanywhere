Claude Code、Codex、Gemini 等多款 AI Agent 的统一远程管理界面。自托管，无需云账号。在手机上随时监控运行于开发机上的 Agent 任务。

## 功能特性

- **多 Provider 支持** — Claude Code、Codex、Gemini、OpenCode、Ollama，一个界面管理所有 Agent
- **自定义 API 端点** — 通过 `~/.claude/settings.json` 将 Claude Code 路由到任意 Anthropic 兼容 API（智谱 GLM、DeepSeek、Ollama 等）
- **会话互操作** — 查看并恢复在 CLI、VS Code 或其他工具中启动的会话，无需新建数据库，直接复用 CLI 持久化数据
- **文件上传** — 从手机相册直接分享截图、照片、PDF 和代码文件
- **推送通知** — Agent 需要审批时即时提醒，锁屏状态下即可响应
- **端到端加密远程访问** — 通过免费中继从任意位置连接，数据全程加密（SRP-6a + TweetNaCl）
- **会话分支** — 从任意消息节点分叉对话，探索不同方案
- **分级收件箱** — 需要关注 → 活跃 → 最近 → 未读，告别终端标签页来回切换
- **全局活动流** — 一览所有 Agent 在所有会话中的实时状态
- **远程设备控制** — 通过 WebRTC 将 Android 模拟器和设备串流到手机，支持触摸输入、导航按键、自适应画质
- **服务端进程管理** — 客户端断开连接不影响 Agent 执行
- **语音输入** — 通过浏览器语音 API 与 Agent 对话
- **移动端优化** — 语法高亮和 Markdown 渲染在服务端完成，移动端体验流畅

无数据库、无云服务、无账号。100% 开源（MIT 协议）。

### 自定义 Claude 端点

编辑 `~/.claude/settings.json`，即可将 Claude Code 路由到任意兼容 Anthropic Messages API 的服务：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "你的 API 密钥",
    "ANTHROPIC_MODEL": "glm-5.1",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-5.1",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5.1",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-5.1"
  }
}
```

完整配置说明请参阅 [自定义端点配置指南](docs/guides/custom-claude-endpoint.md)。

## 快速开始

只要你能安装 Claude Code 或 Codex，就能安装 Yep Anywhere。依赖极少。

```
npm i -g yepanywhere
yepanywhere
```

或从源码构建：
```bash
git clone https://github.com/kzahel/yepanywhere.git
cd yepanywhere
pnpm install
pnpm build
pnpm start
```

在浏览器中打开 http://localhost:3400，应用会自动检测已安装的 CLI Agent。

## 远程访问

**最简方式：** 使用免费公共中继 — 在设置中配置，或通过 CLI 命令完成无头配置：

```bash
yepanywhere --setup-remote-access --username myserver --password "secretpass123"
```

然后从任意位置通过 [yepanywhere.com/remote](https://yepanywhere.com/remote) 连接。

所有流量端到端加密，我们无法看到你的数据。无需注册账号。

**自托管：** 希望运行自己的基础设施？可使用 Tailscale、Caddy 或任何带 SSL 终止的反向代理。详见 [远程访问文档](docs/project/remote-access.md)。

## 为什么不直接用终端？

你*可以*在手机上用终端 — 但等宽字体在小屏幕上体验很差，无法上传文件、没有推送通知、也看不到所有会话的全局视图。Yep Anywhere 提供了完善的 UI，同时保持完全自托管，代码始终在本地运行。

## TOS 合规

Yep Anywhere 使用 Anthropic 官方发布的 [`@anthropic-ai/claude-agent-sdk`](https://www.npmjs.com/package/@anthropic-ai/claude-agent-sdk)。我们不处理认证、不伪造请求头、不篡改 OAuth 令牌。你通过自己的 Claude CLI 完成认证 — 我们只是你的会话的远程界面。

了解更多：[SDK 使用说明](https://yepanywhere.com/tos-compliance.html) | [2026年2月认证说明](https://yepanywhere.com/sdk-auth-clarification.html)
